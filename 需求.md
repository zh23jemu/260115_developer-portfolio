# 个人作品集网站 PRD

## 核心功能
1. 首页：大标题 + 简介 + 头像
2. 关于我：详细介绍 + 技能列表
3. 项目展示：项目卡片列表，每个卡片包含项目名称、项目截图、简短描述、技术栈、项目链接
4. 联系方式：邮箱、GitHub、社交媒体链接

## 设计要求
- 简洁现代的设计风格
- 深色主题（背景 #0a0a0a，文字 #ffffff）
- 使用渐变色作为强调色（主色调推荐紫蓝渐变：linear-gradient(135deg, #667eea 0%, #764ba2 100%)）
- 流畅的滚动动画
- 移动端适配（响应式设计）

## 技术栈

### 前端框架
- **React 18+** - 使用函数式组件和 Hooks
- **TypeScript** - 严格模式，提供类型安全
- **Vite** - 快速的构建工具

### UI 组件库
- **Tailwind CSS** - 实用优先的 CSS 框架
- 或使用 **TDesign / shadcn/ui** 组件库（可选）

### 动画库
- **Framer Motion** - 流畅的滚动动画和交互动画
- 或 **AOS (Animate On Scroll)** - 简单的滚动动画库

### 工具库
- **React Router DOM** - 页面路由
- **React Intersection Observer** - 懒加载和滚动检测

### 图标库
- **Lucide React** - 现代轻量级图标库
- 或 **React Icons** - 丰富的图标集合

### 开发工具
- **ESLint** - 代码规范检查
- **Prettier** - 代码格式化
- **Husky + lint-staged** - Git 提交钩子

## 开发规范

### 代码规范
1. **命名规范**
   - 组件文件：PascalCase（如 `HeroSection.tsx`）
   - 工具文件：camelCase（如 `utils.ts`）
   - 常量：UPPER_SNAKE_CASE（如 `API_BASE_URL`）
   - 类型定义：PascalCase（如 `ProjectItem`）

2. **组件结构**
   ```tsx
   // 1. 导入（外部库 → 内部模块）
   // 2. 类型定义
   // 3. 组件定义
   // 4. 默认导出
   ```

3. **TypeScript 规范**
   - 所有组件必须显式声明 props 类型
   - 避免使用 `any` 类型
   - 使用接口或类型别名定义数据模型

4. **React 最佳实践**
   - 优先使用函数式组件和 Hooks
   - 使用 `useCallback`、`useMemo` 优化性能
   - 组件拆分原则：单一职责，控制在 200 行以内
   - 合理使用 `key` 属性

### 样式规范
1. **Tailwind CSS**
   - 优先使用 Tailwind 工具类
   - 自定义样式写在 `tailwind.config.js`
   - 响应式设计优先使用 `md:`、`lg:` 断点

2. **样式组织**
   ```tsx
   // Tailwind 类名顺序：
   // 布局 → 间距 → 尺寸 → 颜色 → 排版 → 效果 → 其他
   ```

3. **深色主题**
   - 统一使用 Tailwind 的 `dark:` 修饰符
   - 或通过 CSS 变量管理主题色

### Git 提交规范
遵循 **Conventional Commits 1.0.0** 规范：

```
<type>(<scope>): <description>

[optional body]

[optional footer]
```

**Type 类型：**
- `feat`: 新功能
- `fix`: Bug 修复
- `docs`: 文档更新
- `style`: 代码格式调整（不影响逻辑）
- `refactor`: 重构
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建/工具配置
- `ci`: CI/CD 配置

**示例：**
- `feat(hero): 添加头像动画效果`
- `fix(projects): 修复项目卡片在移动端的布局问题`
- `style(lint): 修正 ESLint 警告`
- `docs(readme): 更新安装说明`

### 文件组织结构
```
src/
├── assets/          # 静态资源（图片、字体等）
├── components/      # 可复用组件
│   ├── common/     # 通用组件（按钮、卡片等）
│   └── layout/     # 布局组件（导航栏、页脚等）
├── pages/          # 页面组件
│   ├── Home.tsx
│   ├── About.tsx
│   ├── Projects.tsx
│   └── Contact.tsx
├── hooks/          # 自定义 Hooks
├── utils/          # 工具函数
├── types/          # TypeScript 类型定义
├── styles/         # 全局样式
├── data/           # 静态数据（项目列表、技能等）
└── main.tsx        # 应用入口
```

### 性能优化要求
1. **代码分割**
   - 使用 React.lazy() 和 Suspense 实现路由级代码分割

2. **图片优化**
   - 使用 Next.js Image 组件或懒加载库
   - 优先使用 WebP 格式
   - 提供合适的 `alt` 属性

3. **首屏加载**
   - 关键资源预加载
   - 非关键资源延迟加载
   - 目标：LCP < 2.5s

### 浏览器兼容性
- Chrome/Edge: 最新版本
- Firefox: 最新版本
- Safari: 14+
- 移动端浏览器: iOS Safari 14+, Android Chrome 90+

### 安全要求
1. 所有外部链接使用 `rel="noopener noreferrer"`
2. 输入内容进行必要的转义和验证
3. 使用 HTTPS 部署

## 部署要求
1. **构建输出**：静态 HTML/CSS/JS 文件
2. **部署平台推荐**：
   - Vercel（推荐，零配置）
   - Netlify
   - GitHub Pages
3. **SEO 优化**：
   - 配置合适的 meta 标签
   - 添加 Open Graph 标签
   - 生成 sitemap.xml

## 注意事项
- 保持设计简洁，不要过度设计
- 性能优化：图片使用懒加载
- 确保所有链接可点击
- 代码注释：复杂逻辑必须添加注释
- 定期更新依赖包版本
